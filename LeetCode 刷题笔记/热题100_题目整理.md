# [49] 字母异位词分组

> 难度：中等  
> 标签：`数组` `哈希表`        
> 考点：  
>   1. 通过 `sorted()` 方法对单词的字母进行排序  
>   2. 通过字典实现分组

## 题目描述
给你一个字符串数组，请你将 字母异位词 组合在一起。可以按任意顺序返回结果列表。
示例 1:
输入: strs = ["eat", "tea", "tan", "ate", "nat", "bat"]
输出: [["bat"],["nat","tan"],["ate","eat","tea"]]

**复杂度分析：**  
- 时间复杂度：O(N * KlogK)  
- 空间复杂度：O(N * K)

## 代码实现

```python
class Solution(object):
    def groupAnagrams(self, strs):
        """
        :type strs: List[str]
        :rtype: List[List[str]]
        """
        flag = {}
        # value_list = []
        for words in strs:
            sorted_str = "".join(sorted(words))
            char_list = []

            # 遍历列表，每个元素单词进行字母的排序后作为字典的key
            # 原单词作为value列表里面的元素
            flag.setdefault(sorted_str,[])
            flag[sorted_str].append("".join(words))
            # flag.setdefault(key, []) 只有在 Key 不存在时，才会将其设置为 []。如果 Key 已经存在，它会保留原有的列表，不会进行覆盖。（dict.setdefault()方法的优势）
        return list(flag.values())
```

## 注意：
flag["".join(sorted(char_list))] = value_list.append("".join(char_list))    
不能这样写是因为list.append() 是直接在原列表的内存地址上进行修改，并没有创建一个新的列表对象。返回值是None


# [128] 最长的连续序列

> 难度：中等  
> 标签：`数组` `哈希表`    
> 考点：    
>   1. 将数组转换成哈希集合set,目的一个是去重，一个是in方法判断集合中是否有某元素的时间复杂度为O(1)
>   2. 集合中某元素的前一个数不在集合中，说明此元素可以作为连续序列的起点

## 题目描述
给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。
请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

示例 1：
输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。

**复杂度分析：**  
- 时间复杂度：O(N * KlogK)  
- 空间复杂度：O(N * K)

## 代码实现

```python
class Solution(object):
    def longestConsecutive(self, nums):
        """
        :type nums: List[int]
        :rtype: int
        """
        # 哈希集合中进行 in 查找的时间复杂度是 O(1)
        # 而且也进行了去重
        nums = set(nums)
        max_count = 0
        for i in nums:
            count = 1
            
            if not nums: return 0  # 空列表判断
            # 如果一个数字是起点，那么它的前一个数字（i - 1）应该不在集合中。
            if (i - 1) not in nums:
                min_num = i
                print("min_num",min_num)
            
                while( (min_num + 1) in nums):
                    count += 1
                    print("count:",count)
                    min_num = min_num + 1

                if count > max_count:
                    max_count = count
                print("max_count",max_count)
        return max_count
```

## 注意：

1. 时间复杂度：O(N)  
nums = set(nums)：将列表转换为集合需要遍历一次所有元素，时间复杂度为 O(N)。
for 循环与 while 循环：虽然代码中有嵌套循环，但由于 if (i - 1) not in nums 的限制，while 循环只会在连续序列的起点触发。
这意味着，集合中的每个数字最多被外层 for 循环访问一次，最多被内层 while 循环访问一次。没有重复计算，因此总的时间复杂度为 O(N)。

2. 空间复杂度：O(N)  
nums = set(nums)：我们创建了一个哈希集合来存储数组中的所有元素。在最坏的情况下（数组中没有重复元素），集合需要存储 N 个元素，因此空间复杂度为 O(N)。
